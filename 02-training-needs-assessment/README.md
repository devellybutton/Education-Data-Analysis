## 1. 프로젝트 개요

### 배경  
- 스마트팩토리 전환을 추진 중인 제조 기업을 대상으로  
맞춤형 교육을 운영하던 과정에서 **공통 교육과정 개설 필요성**이 발생했다.  
- 그러나 과정 우선순위를 판단할 **정량적 기준이 없는 상태**였다.

### 목표  
- 기업 HRD 담당자 설문 데이터를 기반으로  
교육 과정별 **중요도와 실행도 구조화**,  
그리고 **공통 교육과정 개설 우선순위 도출**을 한다.

### 분석 대상  
- 제조 기업 **HRD 담당자 설문 응답** 60건

## 2. 문제 정의 및 분석 관점

스마트팩토리 전환 기업을 대상으로 공통 교육과정을 설계해야 하는 상황에서 <br/> 
다음과 같은 **운영 의사결정 문제**가 존재했다.

- 어떤 교육 과정을 **우선적으로 개설해야 하는가**  
- 현장 요구와 현재 실행 수준 간 **차이는 어디에서 발생하는가**  
- 중요하게 인식되지만 충분히 제공되지 않는 과정은 무엇인가  

본 분석은 교육 과정별 <b>중요도(Importance)</b>와 <b>실행도(Performance)</b>를 기준으로  
요구 구조를 정량적으로 비교하고, **중요도–실행도 GAP**을 기반으로  
**공통 교육과정 개설 우선순위 도출**을 목표로 했다.

분석에는 다음 기준을 적용했다.

- 중요도 및 실행도 점수 기반 정량 구조화
- 중요도–실행도(GAP) 계산
- GAP 패턴을 기준으로 교육 과정 유형 분류  
- 유형별 우선순위 도출

## 3. 분석/검증

본 분석은 다음 절차에 따라 진행함.

### 1) 지표 정의

교육 과정별 요구 수준을 정량적으로 비교하기 위해 핵심 지표를 정의함.

- 중요도(Importance) 평균
- 실행도(Performance) 평균
- **GAP (중요도 – 실행도)**: 중요도 대비 실행 부족 정도를 나타내는 핵심 지표

<br/>

### 2) 과정별 통계 산출

설문 데이터를 과정 단위로 요약함.

- 과정별 중요도 평균 계산  
- 과정별 실행도 평균 계산  
- 과정별 GAP 계산  

<br/>

### 3) 운영 전략 분류 기준 설정

중요도, 실행도, GAP을 결합하여 과정 운영 방향을 구분함.

<br/>

### 판단 기준

| 분류 | 판단 기준 | 해석 |
|------|------------|------|
| **우선 개설** | GAP ≥ 0.4 & 중요도 평균 ≥ 3.0 | 중요도는 높으나 실행 수준이 낮은 영역 → 신규 교육 개설 우선 검토 |
| **보완 필요** | GAP ≥ 0.4 & 중요도 평균 < 3.0 | 실행 부족 → 내용 및 운영 방식 개선 필요 |
| **안정 운영** | GAP < 0.2 & 실행도 평균 ≥ 3.0 | 전반적으로 안정 → 현행 과정 유지 가능 |
| **수요 낮음** | GAP < 0.2 & 중요도 평균 < 2.0 | 필요성 및 활용도 모두 낮음 → 운영 축소 또는 제외 검토 |
| **검토 필요** | 기타 영역 | 판단 애매 → 추가 분석 필요 |

<br/>

- 상위 5개 과정 → 우선 개설 필요 과정으로 분류됨.
- 나머지 5개 과정 → 보완 필요 과정으로 분류됨.

<br/>

### 5) 분석 구조 단순화 전략

기존 IPGA 모델을 기반으로 분석 구조를 설계하되,  
의사결정 관점에서 해석의 직관성을 높이기 위해 시각화 구조를 단순화함.

- **중요도 및 실행도** → 2차원 좌표 기반 표현
- **GAP** → 색상 기반 신호로 표현

핵심 신호를 빠르게 인지할 수 있는 형태로 정리함.

## 4. 인사이트 해석

| 구분 | 관찰 결과 | 인사이트 |
|------|-------------|-------------|
| **우선순위 판단 기준** | 중요도와 실행도가 모두 높은 과정보다 GAP이 큰 과정이 상위에 위치함 | 과정 중요도 자체보다 중요 대비 실행 부족 여부가 더 중요한 판단 기준으로 작용함 |
| **중요도–실행도 관계** | 중요도는 높지만 실행이 부족한 과정, 실행은 높지만 중요도 인식이 낮은 과정이 함께 존재함 | 과정 수의 문제가 아니라 현장 인식과 교육 운영 간 차이 가능성 확인 |
| **보완 필요 과정 특성** | 중요도 인식과 실행 수준이 모두 낮은 과정 존재 | 과정 필요성 부족보다는 과정 설계 또는 운영 방식 개선 필요 신호로 해석 가능 |
| **평균 점수 해석 한계** | 동일 평균 점수 과정 간 운영 전략 차이 가능성 존재 | 평균 기반 판단만으로는 의사결정 리스크 존재 |
| **기업 간 응답 차이** | 일부 과정에서 응답 편차 관찰됨 | 기업 특성에 따른 요구 차이 가능성 존재 |
